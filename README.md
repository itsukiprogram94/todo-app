# モダンTodoアプリケーション

##  アプリケーションの概要
「既存のTodoアプリは多機能すぎて複雑、あるいはシンプルすぎて要件を満たせない」という課題から生まれた、**シンプルで直感的、使いやすく美しいUI/UX**を追求したタスク管理アプリケーションです。

React (フロントエンド) と NestJS + GraphQL (バックエンド)、Prisma (データベース) というモダンなフルスタック環境で構築されています。

###  開発の目的とターゲット
日常の買い物、学校の課題、システム開発など、異なるコンテキスト（文脈）を持つタスクを一元管理しつつも、それらが混ざり合って思考のノイズにならないよう設計しています。

**【主なユースケース】**
- 思いついたタスクを即座に登録できる軽量な動作。
- 「タグ機能」により、日常・学校・開発などのタスクを明確に分離。
- 状況に合わせて「未完了タスク」「特定のタグのタスク」など、見え方を瞬時に切り替え可能。

## ✨ 主な機能とCRUD操作
要件定義における「フェーズ1（基礎機能）」および「フェーズ2（実用性の向上）」の主要機能を実装しています。

- **Todoの管理 (CRUD)**
  - `Create`: タスクの新規登録
  - `Read`: 登録されたタスクの一覧表示
  - `Update`: タスク名のインライン編集、完了/未完了ステータスの切り替え
  - `Delete`: 不要になったタスクの削除
- **タグの管理 (CRUD) と振り分け**
  - タグの新規作成、一覧表示、名前とカラーコード（色）の編集、削除。
  - タスクに対して任意のタグを紐づけ可能。
- **即時フィルタリング機能**
  - 「ステータス（完了/未完了）」と「タグ」の掛け合わせによるタスクの絞り込み表示。

## ローカル環境での起動方法
本アプリケーションをローカル環境で動かすための手順です。

### 1. バックエンドの起動
ターミナルを開き、バックエンドのディレクトリで以下のコマンドを実行します。
```bash
# ディレクトリに移動
cd backend

# 依存パッケージのインストール
npm install

# Prismaによるデータベースのセットアップとクライアント生成
npx prisma db push
npx prisma generate

# 開発サーバーの起動 (デフォルト: http://localhost:3000)
npm run start:dev
```

### 2. フロントエンドの起動
ターミナルを開き、フロントエンドのディレクトリで以下のコマンドを実行します

```bash
# ディレクトリに移動
cd frontend

# 依存パッケージのインストール
npm install

# 開発サーバーの起動 (デフォルト: http://localhost:5173 など)
npm run dev
```
